# Marketing Intelligence Agent Evaluation Checklist

## Purpose
This checklist restates the publicly visible evaluation requirements for the G42 `Marketing Intelligence Agent` submission in concise, procurement-friendly form.

## Source Basis
- Public G42 announcement on AI agent recruitment and evaluation gates
- Public job mirror for `Marketing Intelligence Agent`, Abu Dhabi, UAE, `reqId: 2704`
- Repository evidence provided in this workspace

## Evaluation Checklist

### Submission completeness
- Provide an agent profile with specialization and core capabilities
- Provide technical architecture with LLM, system design, and boundary conditions
- Provide performance metrics and demonstrated outcomes
- Provide evidence of prior deployments and reference organizations where available
- Provide agent resume and required supporting submission files
- Provide video demonstration, client references, or testimonials where available
- Ensure the submission is complete; incomplete submissions do not proceed

### Functional fit for the role
- Demonstrate capability across research and planning
- Demonstrate capability across creative and media strategy
- Demonstrate capability across content production
- Demonstrate capability across media management
- Demonstrate capability across measurement and performance analytics
- Demonstrate capability across targeted marketing and audience segmentation

### Technical capability requirements
- Show live production deployment within enterprise environments
- Show compatibility with CRM, marketing automation, analytics, CMS, or media buying platforms
- Show cloud-agnostic architecture
- Show Azure enterprise compatibility without material re-engineering
- Show ability to integrate with sovereign LLM infrastructure through secure API calls
- Show structured campaign data processing and reporting
- Show explainability and audit traceability
- Show defined exception handling and escalation logic

### Security and Responsible AI
- Show robust customer and audience data handling controls
- Show role-based access control
- Show secure authentication protocols
- Show end-to-end audit traceability
- Show documented Responsible AI compliance
- Show bias testing approach and mitigation controls

### Operational maturity
- Show measurable campaign efficiency improvement or time savings
- Show scenario-based performance benchmarks
- Show latency metrics
- Show reliability metrics
- Show known failure modes and mitigation strategies
- Show version history or lifecycle maturity
- Show enterprise integration references
- Demonstrate production-grade maturity, not a prototype or demo-only artifact

### Evaluation gates after submission
- Technical architecture validation
- Empirical performance benchmarking
- Security assessment
- Responsible AI assessment
- Reliability testing
- Stress testing
- Controlled probationary deployment before scaled authorization

## Submission Readiness Interpretation
- `Ready`: Requirement is directly evidenced in the attached documents
- `Partial`: Requirement is addressed, but deployment-specific proof needs a named customer, production metric, or external attachment
- `Placeholder required`: Fact is not available in the workspace and must be supplied by the developer before submission

## Assumptions
- The official G42 job page was not available in the workspace; this checklist is derived from a public job mirror plus G42's public announcement of the AI agent hiring framework.
- Where the role calls for external proof such as video demonstration, client references, named production environments, or contractual SLAs, those items are treated as required attachments outside the code repository unless explicitly provided.
- Checklist revision: April 2026 (aligned with `submission` package `v1.0-candidate`).
